# 当前协作目标画像

仅记录本项目对话中已经明确的稳定目标，用于生成定向建议；不要据此推测用户身份或未表达偏好。

## 已确认目标

- 建立每日自动发现 GitHub 优质项目的长期工作流。
- 同时关注新项目、经典项目、近期活跃项目和经多日数据验证的 star 上升项目。
- 将关键 README、文档、源码、提交信息和校验清单冷存到本地。
- 每个正式项目形成约一万字、以一手材料为基础的中文深度分析。
- 清楚回答项目做了什么、关键机制、值得借鉴之处与不宜照搬之处。
- 对外部内容先做来源充分性检查；不足时降级，不伪装成长文总结。
- 长文与研究输出版本化，保留版本说明，方便比较与回退。
- 自动流程应使用一致、严格、可审计的筛选标准，保证每日输出水平相当。
- 借鉴建议需要结合本 GitHub 雷达的真实建设阶段，提供可判断的具体选项，但保留用户最终决定权。
- 本地成果会持续同步到公开仓库 `yihan498/github-project-radar`。

## 当前系统状态

- 已有 GitHub API 候选发现、多日 star 观测、冷存、Skill、定时任务与首篇 Engram 样稿。
- 已建立来源充分性、事实/推断分离和增长证据边界。
- 当前提升重点是评分一致性、成稿质量门槛和定向经验迁移。

## 定向建议优先方向

优先评价某个项目能否改进：候选发现、趋势识别、证据冷存、评价校准、长文生产、质量验证、版本管理、自动任务可靠性、公开仓库呈现。超出这些目标的建议放入“探索性”，不得强行写成核心建议。

每次正式推荐结束后，只在用户明确表达新目标或工作流发生实质变化时更新本文件。
