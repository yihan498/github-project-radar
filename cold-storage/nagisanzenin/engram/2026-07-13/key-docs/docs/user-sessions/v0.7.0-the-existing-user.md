# User session report — v0.7.0

```
topic: "what does /coach look like the day I update?"   mode: existing-user upgrade
real minutes: ~15        nodes encoded: 0        reviews cleared: 0
state: a read-only copy of the founder's REAL ~/.claude/learning (7 encoded, 0 reviewed)
```

**Why this session and not a learning session.** v0.7's engine changes are read-only, but one of
them lands on *every existing user's screen the moment they update*: `grader_unvalidated` is
`true` by default, and it stamps the retention figure. Five hundred people installed this thing.
The question that matters is not *"is the audit correct"* — the other six gates answer that. It
is *"what does a stranger see on the first day, and does it drive them away?"*

---

## WHAT WORKED

- `grader-health` on a state that has never been audited returns `unaudited` and says why, in one
  calm paragraph with a next step. It does not error, does not block, does not nag.
- The whole `/coach audit` flow ran end to end against the **real** assessor: `gold` → three
  independent blind runs → `assessor-audit`. Every one of the 66 `sid`s round-tripped. The
  measurement is real and it is **QWK 0.93**.
- A v0.6 state directory with no `audits/` and no `gold/` upgraded silently. Nothing to migrate.
- `doctor` stayed `ok: true` throughout.

## WHAT CONFUSED ME

- **`/coach` now had two "you have failed" messages stacked on top of each other.** First
  *"THE LOOP HAS NEVER CLOSED: 7 concepts came due and none were reviewed"*, then immediately
  *"the grader that writes every receipt has never itself been graded… every number it feeds is
  unearned."* Both true. Together, on the first screen after an update, they read as a system
  that is disappointed in me.

## WHAT ANNOYED ME — and what I would have quit over

- **This line, exactly as it shipped in the branch:**

  ```
  [grader unaudited — QWK unknown; run /coach audit] insufficient-data (no reviews yet)
  ```

  **It is a caveat on a number that does not exist.** There are no grades to distrust, because
  there are no retrievals. It is a disclaimer for a measurement nobody made, and it is stapled to
  the front of a line whose entire content is *"there is nothing here yet."*

  I would not have quit over it. I would have stopped reading the output — which is the same
  thing, more slowly. `docs/05` P14 is explicit that a wall of debt is the churn *trigger*, not
  the cure, and this release quietly built a second brick.

  **Fixed before ship, both halves:** the engine stamps the read only when a retention figure
  actually exists (the flag stays `true` in the payload — it is a true fact — but the narrator is
  not handed a disclaimer for nothing), and `/coach` now **skips the whole oracle section when
  `loop_closure.rate == 0`**, under the same doctrine that already governs everything else on that
  screen: *when the loop has never closed, nothing below it is real, so say the one thing that
  matters and stop.*

## WHAT IT TOLD ME vs WHAT WAS TRUE

Every number, checked by hand, on one state, side by side (§4.8 Q1):

| it said | is it true? | does it flatter? |
|---|---|---|
| `loop_closure 0/7` | yes — 7 encoded, 0 reviewed | no. It is the least flattering true thing the system can say, and it is the whole point |
| `decay: 2.7 of 7 survive 30 days; 5.6 if reviewed today` | yes — reproduces `docs/08` exactly from live state | no |
| `retention: insufficient-data … 7 past due and unretrieved (~70% now)` | yes | no — the debt is stated, not hidden |
| `grader-health: unaudited` | yes | **no, and this is the one that could have gone the other way.** It defaults to *unvalidated*, never to *fine* |
| `grader-health: pass, QWK 0.93` (after the audit) | yes | **no — and I checked the direction.** 0 of 198 judgments graded UP |

`grader-health`, `stats.grader_health`, `stats.retention` and `retention` all report the same
verdict on the same state. They agree because they are one function; they were checked anyway,
because v0.6.2 shipped two commands that contradicted each other and nobody had ever run them
side by side.

**And the one I got wrong until an outsider looked:** the HTML dashboard rendered a full-width
green *"30-day recall 100%"* bar from a grader that had **failed its own audit**, with the word
`grader` appearing **zero times** on the page. I read the JSON five different ways and never
opened the file. That is now §4.8 Q4, in writing, with a `grep`.

## WOULD A STRANGER GET THROUGH THIS?

**Yes — now.** Before the two fixes above, they would have got through it and quietly stopped
reading, which is worse than a crash, because a crash gets reported.

The honest limit of this session: **I cannot feel what it is like to come back after three days.**
`ENGRAM_TODAY` moves the clock, not the person. The founder's own account — 7 encoded, 0 reviewed
— is still the loudest fact in this repository, and v0.7 did not change it. It only means that
when the review finally happens, the number it produces will have been written by a grader that
somebody checked.

## VERDICT

**SHIP.**
